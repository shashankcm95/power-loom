# Shared Knowledge Base — Agent Team

Filesystem-based, content-addressed knowledge base for HETS. Implements the [Shared Knowledge Base](../patterns/shared-knowledge-base.md) and [Content-Addressed References](../patterns/content-addressed-refs.md) patterns.

## Layout

```
kb/
├── README.md          (this file)
├── manifest.json      (machine-readable index — generated by `kb-resolver scan`)
├── <topic>/<doc>.md   (KB docs with frontmatter)
```

## Doc format

Every KB doc has frontmatter:

```yaml
---
kb_id: <topic>/<doc-name>     # stable, MUST match the file path
version: 1                     # bump on substantive edits
tags: [topic-tag, scope-tag]   # for catalog query
---

## Summary (≤5 lines, distilled)

[Card form. Cheap to paste inline in spawn prompts.]

## Full content

[Long form. Loaded on demand via `kb-resolver cat <kb_id>`.]
```

## Resolver CLI

`scripts/agent-team/kb-resolver.js` — invoke via `node`:

| Subcommand | Purpose | Example |
|------------|---------|---------|
| `cat <kb_id>` | Print doc body | `node kb-resolver.js cat hets/spawn-conventions` |
| `hash <kb_id>` | SHA-256 of body | `node kb-resolver.js hash hets/spawn-conventions` |
| `list [--tag X]` | List manifest entries | `node kb-resolver.js list --tag hets` |
| `resolve <ref>` | Resolve a ref string | `node kb-resolver.js resolve kb:hets/spawn-conventions@a3f1b2c4` |
| `scan` | Walk tree, refresh manifest | `node kb-resolver.js scan` |
| `snapshot <run-id>` | Freeze manifest into run-state | `node kb-resolver.js snapshot chaos-20260502-080000` |
| `register <kb_id>` | Single-file update (alt to scan) | `node kb-resolver.js register web-dev/react-essentials` |

## Reference syntax

- `kb:<topic>/<doc>` — current version (resolves to whatever the manifest says now)
- `kb:<topic>/<doc>@<8-char-hash>` — snapshot-pinned; resolver rejects with `hash_mismatch` if doc has changed

## Workflow for HETS runs

1. **Run start**: orchestrator calls `kb-resolver snapshot <run-id>` — freezes the manifest into `swarm/run-state/<run-id>/kb-snapshot.json`
2. **Spawn time**: spawn prompts include refs from the snapshot, e.g., `your KB scope: kb:hets/spawn-conventions@a3f1b2c4`
3. **Agent first action**: resolve refs via `kb-resolver resolve <ref>` to load the docs the agent needs
4. **KB edited mid-run**: snapshot is unchanged; in-flight agents continue resolving against the frozen view

## Adding a KB doc

1. Create `<topic>/<doc-name>.md` under `kb/` with the frontmatter format above
2. Run `kb-resolver scan` (or `register <kb_id>` for a single new file)
3. Verify with `kb-resolver list`

## Path resolution

- Default location: `~/Documents/claude-toolkit/skills/agent-team/kb/`
- Override via env var: `HETS_KB_DIR=<path>`
- Resolver also reads `HETS_RUN_STATE_DIR` for snapshot writes (default: `~/Documents/claude-toolkit/swarm/run-state/`)
