# H.9.1 — shellcheck in local verification harness

## Context

H.9.0 closed the layer-mismatch gap for markdown content-format discipline (markdownlint was in CI but not local; CI sole enforcer; 49-commit HT track accumulated 22 violations). H.9.1 addresses the sibling gap for bash content-format discipline.

### The sibling gap is broader than H.9.0's

- **H.9.0 gap shape**: lint existed in CI (`.github/workflows/ci.yml` Markdown lint job), absent from local smoke. Per-phase pre-approval gates reviewed sub-plan + implementation but NOT cutover-time bookkeeping; verification probes were code/test focused.
- **H.9.1 gap shape**: shellcheck existed NEITHER in CI NOR in local smoke. Bash content-format discipline was unenforced at any layer. The substrate has 10 `*.sh` files (install.sh + bin/ + tests/ + scripts/) accumulating an untracked baseline of 98 total findings (78 at error severity, 20 at info/warning level).

H.9.1 establishes parity at the local layer for shellcheck at error severity. CI parity (separate concern) deferred to drift-note candidate; per H.9.0 precedent, local-layer enforcement is the substrate's leading edge.

### Empirical pre-validation

Baseline shellcheck error-severity run at H.9.1 implementation start (`npx --yes shellcheck --severity=error` against 10 substrate `*.sh` files):

| Rule | Count | Files |
|------|-------|-------|
| SC2168 (`local` outside function) | 73 | `tests/smoke-h{4,7,8}.sh` |
| SC2148 (missing shebang) | 4 | `tests/smoke-h{4,7,8,ht}.sh` |
| **Total** | **78** | **4 test files** |

Other substrate `*.sh` files (`install.sh`, `bin/migrate-to-plugin.sh`, `scripts/*.sh`) are shellcheck-clean at error severity (only info/warning-level findings remain, which are below the H.9.1 gate threshold).

### Root cause: shellcheck cannot follow `install.sh` source path

The 4 affected test files are SOURCED by `install.sh run_smoke_tests()` function (verified at install.sh:231-234: `source "$SCRIPT_DIR/tests/smoke-h{4,7,8,ht}.sh"`). At runtime, `local` IS inside the `run_smoke_tests()` function scope — confirmed by each test file's header: "DO NOT execute directly — depends on parent-scope `local passed`, `local failed`".

Shellcheck without source-following sees the `*.sh` files as standalone bash scripts where:
- Missing shebang (SC2148): test files have no shebang because they're sourced, not executed
- `local` outside function (SC2168): test files use `local` at "top-level" in shellcheck's view, but at runtime that's inside the `run_smoke_tests()` body

Both fires are false positives at error severity caused by shellcheck's static-analysis lacking the `install.sh`→`tests/*.sh` source path.

### Why not weaken shellcheck config instead

The config-guard hook discipline (per ADR-0001 + ADR-0003) blocks weakening of lint configs at the substrate root level. H.9.0 set precedent: "fix the content to satisfy existing lint, don't weaken the lint to satisfy the content." H.9.1 inherits this: file-top directives explain WHY the rule doesn't apply (sourced-file context shellcheck can't follow), localized to the 4 affected files — NOT a `.shellcheckrc` config that weakens the rule substrate-wide.

The directive-vs-config distinction matters: a directive documents the engineering context (declarative); a config weakens the rule globally (loses discipline). H.9.1 takes the directive path per H.9.0 precedent.

## Implementation strategy

### Edit 1 — Add `# shellcheck shell=bash` + `# shellcheck disable=SC2168` to 4 test files

Each of `tests/smoke-h{4,7,8,ht}.sh` gets 2 lines added at file top (above the existing first comment line):

```bash
# shellcheck shell=bash
# shellcheck disable=SC2168  # H.9.1 — sourced by install.sh run_smoke_tests(); `local` is function-scope at runtime
```

The 1st directive declares shell (resolves SC2148). The 2nd directive suppresses SC2168 file-wide with inline explanation (resolves 73 SC2168 fires across the 4 files). Total: 4 files × 2 lines = 8 lines added.

### Edit 2 — Add Test 81 to `tests/smoke-ht.sh`

Insert between Test 80 (H.9.0 markdownlint) and Test 65 (H.8.7 trailer) — same position as Test 80 followed H.7.27 chronologically. Test 81 invocation uses `find` + `xargs` for future-extensibility (new `*.sh` additions get checked automatically):

```bash
echo -n "  Test 81 (H.9.1 shellcheck error-severity in local smoke harness; npx shellcheck against substrate .sh files): "
T81_OUT=$(cd "$SCRIPT_DIR" && find . -name "*.sh" -not -path "./node_modules/*" -not -path "./.git/*" -print0 | xargs -0 npx --yes shellcheck --severity=error 2>&1)
T81_EXIT=$?
if [ $T81_EXIT -eq 0 ]; then
  echo "OK (substrate shellcheck clean at error severity; 0 errors)"
  passed=$((passed + 1))
else
  echo "FAIL: shellcheck reported errors"
  echo "$T81_OUT" | tail -5
  failed=$((failed + 1))
fi
```

First-run on a fresh machine takes ~10-15s to download shellcheck binary into npx cache; subsequent runs ~1-2s.
