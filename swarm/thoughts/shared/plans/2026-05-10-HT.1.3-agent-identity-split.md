## ADR-0002 shape â€” bridge-script entrypoint criterion

**Title**: Bridge-script entrypoint criterion: when multi-responsibility-at-bridge-script accumulates vs when split applies.

**Status**: active (forward-looking; HT.1.3 onward).

**Context**: 5-phase multi-responsibility-at-lifecycle-entrypoint pattern (HT.0.1 hooks: auto-store-enrichment.js / pre-compact-save.js / session-reset.js â€” 3 hooks; HT.0.2 agent-identity.js â€” 1698 LoC; HT.0.3 commands/build-team.md â€” 322 LoC; HT.0.5b SKILL.md â€” 333 LoC; HT.0.7 install.sh `run_smoke_tests` â€” 537 LoC). Substrate accretes multi-responsibility at script entrypoints because each new responsibility adds "one more thing this script does" rather than "split this script."

**Decision**: A bridge-script entrypoint accrues multi-responsibility cleanly when (a) all responsibilities share the same lifecycle phase (e.g., Stop hook fires once, can do N things in sequence) AND (b) the entrypoint surface is bounded (â‰¤800 LoC AND â‰¤5 responsibilities AND â‰¤500 LoC per responsibility). Split applies when EITHER (a) responsibilities cross lifecycle boundaries OR (b) any size bound is breached.

**Consequences**:
- HT.1.3 splits agent-identity.js (1698 LoC Ă— 5 Ă— ~340 LoC avg) â€” both bounds breached
- HT.1.4 splits install.sh `run_smoke_tests` (537 LoC Ă— 4-phase test groupings Ă— ~134 LoC avg) â€” bound (a) holds; bound (b) marginal; phase grouping is the natural extraction shape
- HT.1.5 splits commands/build-team.md (322 LoC Ă— 6 Ă— ~54 LoC avg) â€” neither bound breached individually; multi-responsibility-density-per-LoC is the trigger
- 3 multi-responsibility hooks (HT.0.1) NOT yet split per ADR-0002: each is bounded â‰¤800 LoC AND responsibilities all share lifecycle (Stop / PreCompact / SessionStart) â€” articulate-only is correct

**Files affected**: agent-identity.js, install.sh, commands/build-team.md.
