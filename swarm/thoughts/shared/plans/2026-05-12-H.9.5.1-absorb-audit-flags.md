## Architect FLAGs — disposition

**Source**: architect report. Plan file `swarm/thoughts/shared/plans/2026-05-11-H.9.5-yamllint-frontmatter.md:136,163` explicitly acknowledges that H.9.5 establishes `wrap narrative scalar values in double quotes; use backticks for internal inline-code` as substrate authoring convention in practice but defers codification. Compounding: HT-state.md `last_session_phase_prior: × 8 → last_session_phase_priors: block-list` is an institutional ledger shape rename.

**Disposition (this phase)**: ABSORB via lightweight BACKLOG decision-record entry (5th entry following HT.1.6 + HT.1.12 + HT.1.15 + H.9.0). The entry codifies (a) the narrative-quoting convention; (b) the `last_session_phase_priors:` block-list shape; (c) the migration-script-at-substrate-path + delete-post-ship transparency pattern; (d) cross-references to Test 83 enforcement; (e) explicit non-promotion-to-ADR rationale per HT.0.9-verify FLAG-5 right-sizing.

### MEDIUM — H.9.5 mixed 3 distinct mechanical jobs at one ship

**Source**: architect report. (1) Test 83 gate addition; (2) narrative-quoting migration; (3) duplicate-key consolidation + HT-state.md schema-like rename. Rollback shape was single-`git checkout` reverting all three together.

**Disposition (this phase)**: NOTE — substrate convention learning for H.9.6+ phases (avoid multi-component atomic ships when individual components have distinct institutional weight). No retroactive action; H.9.5 ship is in-tree and verified-clean.

### LOW — Test 83 glob expansion could exceed argv limits at scale

**Source**: architect report. `npx --yes yaml-lint "$T83_TMPDIR"/fm-*.yaml` (line 424 of `tests/smoke-ht.sh`) uses unquoted glob expansion; with 130 substrate files current state OK on macOS default ARG_MAX, but if substrate grows past ~4000 frontmatter files or runs on constrained shell, glob argv could exceed limits. Tests 81/82 use `xargs` for the same reason.

**Disposition (this phase)**: drift-note 77 captures this + the sibling code-reviewer LOW below. Future-proofing fix deferred to H.9.6+ as cheap incidental refactor.

## Code-reviewer FLAGs — disposition

### MEDIUM — backtick substitution applied to natural-language quoted strings, not only inline-code

**Source**: code-reviewer report. Example at `swarm/thoughts/shared/plans/2026-05-11-H.9.0-markdownlint-local-verification.md` methodology field: pre-migration `"**/*.md"` shell-quoted arg → post-migration backtick-wrapped `` `**/*.md` ``. Migration script applied substitution uniformly to all `"..."` inside values without distinguishing inline-code from shell-quoted strings or quoted prose.

**Disposition (this phase)**: NOTE — `parseFrontmatter` strips outer quotes + does not interpret backticks specially; runtime parsed values are identical pre/post-migration. The semantic drift is in archival documentation only, not operational. The codified convention in the new BACKLOG entry documents this: substrate inline-code convention uses backticks; archival prose values get backtick-rendered post-migration. No retroactive fix needed.

### MEDIUM — migration script logic not permanently captured as auditable artifact

**Source**: code-reviewer report. `scripts/h95-fix-frontmatter.js` + `scripts/h95-fix-htstate-dupkeys.js` deleted post-ship; sub-plan captures intent in prose but exact edge-case handling (skip logic for already-YAML-valid forms: empty / inline array / already-quoted / numeric / boolean / null / timestamp / git hash) is not reconstructable.

**Disposition (this phase)**: NOTE — yaml-lint passing at 0 errors post-migration confirms the output is correct regardless of which specific entries got migrated vs skipped. The new BACKLOG entry codifies the migration-script-at-substrate-path + delete-post-ship pattern + recommends future migration scripts include their full transformation logic in the sub-plan prose section (not just intent summary).

### LOW — Test 83 `head -1` check zero-tolerance for files with leading BOM/blank-line

**Source**: code-reviewer report. `head -1 "$T83_FILE" | grep -q "^---$"` correctly filters non-frontmatter files (e.g., `CHANGELOG.md`); however, any `.md` file whose first line is a BOM, blank line, or UTF-8 sequence before `---` would be silently skipped.

**Disposition (this phase)**: drift-note 77 (combined with architect LOW xargs-argv) captures Test 83 robustness future-proofing. No current substrate file fails this case; pure future-proofing.
