# H.9.0 — markdownlint in local verification harness

## Context

Post-HT.3.3 + markdownlint-fix retrospective (2026-05-11) surfaced a structural process gap. Substrate had:

- **Strong design-time discipline** — 7 per-phase pre-approval gates across HT.0.9-verify + HT.1.3 + HT.1.7 + HT.1.13 + HT.2.0 + HT.2.1 + HT.2.3, 100% single-pass FLAG absorption
- **Strong runtime discipline** — 14 hooks (fact-force-gate + config-guard + error-critic + prompt-enrich-trigger + session-self-improve-prompt + session-reset + auto-store-enrichment + console-log-check + pre-compact-save + session-end-nudge + verify-plan-gate + validate-no-bare-secrets + validate-frontmatter-on-skills + validate-plan-schema + validate-kb-doc + validate-adr-drift); 4 ADRs codifying invariants
- **Weak content-format-time discipline** — `.markdownlint.json` config existed since H.7.8 but markdownlint was NOT in the local verification harness. CI was the sole enforcer; CI runs on push to main + PR, not during feature-branch work. The 49-commit HT track (HT.0.9-verify → HT.3.3) accumulated 22 MD037 + MD038 violations across CHANGELOG/SKILL/BACKLOG ledger entries that surfaced only at the final fast-forward merge to main.

### The structural blind spot

Pre-approval gates reviewed sub-plan + implementation. They did NOT review cutover-time bookkeeping (CHANGELOG/SKILL/HT-state.md prose written AFTER gate approval). Verification probes were code/test focused (`install.sh --hooks --test` + `_h70-test.js` + `contracts-validate.js`); markdownlint was absent.

The MD037 trigger pattern (paired `_` markers on a single line forming emphasis with spaced content between) was sneaky: each individual phase entry might add 1-2 `_h70-test*` references; the cumulative effect of 30+ phase entries in CHANGELOG produced many same-line pair sites.

### Self-referential irony

The H.7.27 CHANGELOG entry (BACKLOG.md L275-329) DOCUMENTED the migration of `[MARKDOWN-EMPHASIS-DRIFT]` PostToolUse hook TO markdownlint pipeline as the canonical migration shape. The entry itself contained the literal trigger pattern `(HETS_TOOLKIT_DIR _h70-test _lib/ paragraph)` unwrapped — violating the migration's own discipline. Author-side discipline existed in slopfiles prose but didn't apply to the ledger writing itself.

### Why now (H.9.0 timing)

Per HT.2.5 soak gate readiness assessment, substrate is in post-HT-closure soak period toward v2.0.0. The 2026-05-11 markdownlint fix (`1f00bf1`) was the smallest viable content fix. H.9.0 closes the underlying process gap that allowed the failure to ship: add lint to local verification so the failure cannot recur silently in future ledger writing.

H.9.0 is the natural kickoff for the H.9.x post-HT track — start with the process improvement that the HT track itself missed, before substantive H.9.x trajectory work (per HT.2.5 readout focus areas: error-critic.js fail-soft + 8 atomic-write sites + Atomics.wait + Tier-3 audit findings).

## Implementation strategy

### Edit 1 — Add Test 80 to `tests/smoke-ht.sh`

Insert between Test 79 and Test 65 (Test 65 is out-of-order leftover from H.8.7 era at end of file). Test 80 invocation:

```bash
echo -n "  Test 80 (H.9.0 markdownlint in local smoke harness; 'npx --yes markdownlint-cli2 \"**/*.md\" \"#node_modules\" \"#swarm\"'): "
T80_OUT=$(cd "$SCRIPT_DIR" && npx --yes markdownlint-cli2 "**/*.md" "#node_modules" "#swarm" 2>&1)
T80_EXIT=$?
if [ $T80_EXIT -eq 0 ]; then
  echo "OK (substrate markdown lint clean; 0 errors)"
  passed=$((passed + 1))
else
  echo "FAIL: markdownlint reported errors"
  echo "$T80_OUT" | tail -5
  failed=$((failed + 1))
fi
```

Same command CI uses. First-run on a fresh machine may take ~5-10s to fetch markdownlint-cli2 via npx cache; subsequent runs are ~3s. Acceptable for smoke harness latency.

### Edit 2 — Add lightweight BACKLOG entry for ledger-authoring convention

4th lightweight BACKLOG entry (per HT.1.6 + HT.1.12 + HT.1.15 precedent). Codifies the convention:

- Ledger-shape entries (CHANGELOG.md, SKILL.md phase status, BACKLOG.md, HT-state.md) referencing underscored identifiers (`_h70-test*`, `_lib/*`, `__test_internals__`, `_stripInlineComment`, etc.) wrap them in backticks
- Avoids MD037 emphasis-pair fires when same line has 2+ underscored identifiers
- Documents the local-verification Test 80 as the enforcement mechanism

### Edit 3 — CHANGELOG.md + SKILL.md + HT-state.md cutover

Standard per-phase cutover; H.9.0 entry above HT.3.3.
