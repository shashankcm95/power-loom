# H.9.6.2 â€” Test 83 hardening + diagnostic improvements + drift-note 78

## Context â€” H.9.6.1 hotfix retrospective

H.9.6 cutover edit accidentally re-introduced the duplicate `last_session_phase_priors:` key bug that H.9.5 originally fixed. Local install.sh smoke runs **did not surface the failure cleanly** â€” output was truncated mid-Test 83, exit code was non-zero but with no FAIL message visible. CI cleanly caught it. Hotfix H.9.6.1 (single-line deletion) restored substrate to clean state.

Post-mortem identifies the root cause of the local-smoke false-negative: **bash `set -e` fires on failed command substitution assignment** (`T_OUT=$(failing_cmd 2>&1)`). Empirical reproduction:

```bash
$ cat <<'EOF' | bash
set -euo pipefail
echo "Before"
FOO=$(false)
echo "After"  # never prints
EOF
Before
$ echo $?
1
```

In Tests 80-83, the pattern is:

```bash
T83_OUT=$(npx --yes yaml-lint "$T83_TMPDIR"/fm-*.yaml 2>&1)
T83_EXIT=$?
if [ $T83_EXIT -eq 0 ]; then ...
else
  echo "FAIL: yaml-lint reported frontmatter errors"
  echo "$T83_OUT" | tail -10
  failed=$((failed + 1))
fi
```

When yaml-lint exits non-zero (legitimate failure), `set -e` fires on the `T83_OUT=$(...)` assignment BEFORE `T83_EXIT=$?` can capture the exit code. install.sh exits immediately. The `else` branch never runs. The FAIL diagnostic never prints. `failed=$((failed + 1))` never increments. The `Results: N passed, M failed` line at end of `run_smoke_tests()` never prints.

This is a **diagnostic-surface failure mode**: the test failure is real, but the smoke harness hides it. CI was the only surface that surfaced the real exit code (1).

## Fix shape (per test)

Apply uniform safe pattern across Tests 80, 81, 82, 83:

```bash
# BEFORE (broken under set -e):
T83_OUT=$(npx --yes yaml-lint "$T83_TMPDIR"/fm-*.yaml 2>&1)
T83_EXIT=$?

# AFTER (set -e safe):
T83_EXIT=0
T83_OUT=$(npx --yes yaml-lint "$T83_TMPDIR"/fm-*.yaml 2>&1) || T83_EXIT=$?
```

Rationale: the `|| T83_EXIT=$?` construct creates a `||` list. Per bash documentation `set -e`: *"The shell does not exit ... any command executed in a && or || list except the command following the final && or ||"*. The `T83_OUT=$(...)` assignment is the command BEFORE the `||`; its exit code is captured by the second clause; errexit suppressed. `T83_EXIT` is pre-initialized to 0 so on success (`||` doesn't fire) it keeps the right value.

## Diagnostic improvements

### Tests 80-82 (markdownlint, shellcheck, jq)

- Change `echo "$T_OUT" | tail -5` â†’ `echo "$T_OUT" | head -30`. Lint output emits the failing file path + error context at the TOP of output; `tail -5` clipped that and showed only the trailing stack frames.
- Add `(exit $T_EXIT)` to the FAIL message: surfaces the underlying tool's exit code (some tools emit different codes for different failure modes).

### Test 83 (yaml-lint) â€” additional per-file mapping

yaml-lint's error output references `fm-N.yaml` (the temp file name) but provides no path to the substrate source `.md` that owns the bad frontmatter. The H.9.6.1 post-mortem revealed that diagnosing which substrate file owned the broken YAML required manually re-running the find/awk extractor.

Fix: write a manifest file `$T83_TMPDIR/manifest.txt` mapping each `fm-N.yaml` to its substrate source path. On failure, grep yaml-lint output for `fm-N.yaml` references + cross-reference manifest + print owning source path(s).

```bash
T83_MANIFEST="$T83_TMPDIR/manifest.txt"
: > "$T83_MANIFEST"
while IFS= read -r T83_FILE; do
  if head -1 "$T83_FILE" 2>/dev/null | grep -q "^---$"; then
    T83_COUNT=$((T83_COUNT + 1))
    awk '...' "$T83_FILE" > "$T83_TMPDIR/fm-$T83_COUNT.yaml"
    echo "fm-$T83_COUNT.yaml -> $T83_FILE" >> "$T83_MANIFEST"
  fi
done < <(...)
...
# On failure:
echo "$T83_OUT" | grep -oE 'fm-[0-9]+\.yaml' | sort -u | while IFS= read -r FM; do
  grep "^$FM " "$T83_MANIFEST" || true
done
```

Empirical fault-injection confirms output:

```text
Test 83 ... : FAIL: yaml-lint reported frontmatter errors (exit 1)
  ...yaml-lint error stack...
  Failing frontmatter sources (per manifest):
fm-44.yaml -> /Users/.../swarm/thoughts/shared/HT-state.md
```

Direct path to the broken file. No manual investigation needed.
