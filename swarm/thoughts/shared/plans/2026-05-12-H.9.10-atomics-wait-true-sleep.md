# H.9.10 — `_lib/lock.js` Atomics.wait true-sleep migration

## Context

`scripts/agent-team/_lib/lock.js` is the substrate's file-lock primitive shared by 15 consumers (substrate scripts + 2 hook consumers post-H.9.9). The wait loop at L68-72 uses a CPU-busy-wait spin loop to bridge sub-10ms sleep resolution gap:

```javascript
const end = Date.now() + sleepMs;
// H.9.7: intentional busy-wait under tight ms budget (ADR-0006 invariant 3
// refactor-not-suppress for no-empty rule); JS lacks native sleep < ~10ms
// resolution short of using Atomics.wait (drift-note candidate H.9.10)
while (Date.now() < end) { /* spin */ }
```

The H.9.7 comment explicitly self-identifies this as the H.9.10 drift-note candidate. Substrate's lock-acquire loop iterates `(maxWaitMs / sleepMs) = (3000/50) = 60×` per worst-case timeout. Each iteration: lock-take-attempt (fast) + (if contention) ~50ms CPU spin. Total CPU burn under contention: ~3s of busy-wait per timeout window.

**Drift impact (current state)**:
- Lock contention scenarios (concurrent PostToolUse hooks, paired session-end fires, multi-process kb-resolver writes) burn ~50ms CPU per retry × up to 60 retries = up to 3s CPU per process per timeout
- Tests 79 + 85 (lock-contention validation) exercise this code path — they observe wall-clock elapsed (correct) but the underlying CPU usage is suboptimal
- Substrate-quality regression risk: as more concurrent substrate processes spawn (sub-agent fan-out + parallel verification), aggregate CPU burn during lock-wait grows quadratically

**H.9.10 fix**: replace busy-wait at L68-72 with `Atomics.wait` on `SharedArrayBuffer + Int32Array` — synchronous OS-level sleep; same wall-clock duration; zero CPU during wait. Non-breaking change (lock.js API surface unchanged).

**Why gate INVOKED (not MANDATORY)**:
- ADR-0002 invariant 4: `_lib/*` shared helpers carved out as orthogonal to bridge-script criterion → MANDATORY-gate does NOT apply (HT.2.3 + H.9.8 precedent)
- Gate INVOKED per HT.1.7 + HT.1.13 + HT.2.3 + H.9.8 substrate-`_lib/*`-precedent: option-axis design surface (Atomics.wait API integration) + HIGH-class bug catchable at design (misuse class: SharedArrayBuffer construction errors; Atomics.wait return-value interpretation; thread-safety in single-threaded Node)

## Option-axis (4 options; recommendation Option A)

| Option | Approach | API Change | Pros | Cons |
|--------|----------|------------|------|------|
| **A (recommended)** | `Atomics.wait(int32, 0, 0, sleepMs)` on SharedArrayBuffer | NONE | Synchronous; OS-level sleep; zero CPU; non-breaking; cross-platform; built-in (zero-runtime-dep per ADR-0006 invariant 4) | Requires SharedArrayBuffer construction (one-time setup); throws on non-shared array (mitigate via try/catch or fail-fast assertion) |
| B | `child_process.spawnSync('sleep', [String(sleepMs/1000)])` | NONE | Simple; cross-platform | Subprocess fork per retry (~5-10ms overhead per spawn); CPU cost of fork worse than busy-wait at short sleepMs; system-call-heavy |
| C | Migrate to async API (setTimeout + Promise + await) | BREAKING (all consumers need `await`) | Idiomatic JS; fine-grained control | BREAKING API change; 15 consumer migration scope; out-of-scope for H.9.10 |
| D | Native addon (e.g., libc usleep) | NONE | Best performance | Substrate dependency surface; conflicts with ADR-0006 invariant 4 zero-runtime-dep; cross-platform compilation complexity |

**Resolved**: Option A. Atomics.wait synchronous OS-level sleep; non-breaking; zero-runtime-dep; cross-platform; standard library. Substrate test cohort (Test 79 + Test 85) wall-clock-elapsed-invariant preserved.

```javascript
// Module-level shared-array allocation (one-time per process; reused across
// all lock-wait calls). 4-byte SharedArrayBuffer + Int32Array view satisfies
// Atomics.wait API requirements without allocating per-call.
//
// architect FLAG-4: SharedArrayBuffer is REQUIRED by Atomics.wait's type
// contract (Int32Array view on shared memory). Substrate does NOT share this
// buffer across threads — each worker_threads import would get its own SAB
// via per-thread module cache. Buffer is a scratch slot for the sleep
// primitive; never notified, only used for timeout-based blocking.
//
// architect FLAG-1 + code-reviewer convergent: wrap SAB construction in
// try/catch with busy-wait fallback. If SharedArrayBuffer is unavailable
// (memory-constrained env, --no-shared-array-buffer flag, hardened runtime),
// module load must NOT fail (would break all 15 consumers including 2 hook
// fail-soft consumers per ADR-0001 invariants 2-3). Lazy fallback preserves
// substrate reliability commitment.
//
// architect FLAG-10 + LOW-8: Atomics.wait is JS standard (ECMA-262 §25.4.5);
// V8 normalizes across macOS pthread + Linux futex implementations. OS
// scheduler granularity ~1-2ms per Atomics.wait call; accumulates to
// ~60-120ms drift over worst-case 60-iteration timeout. Tests 79 (1-5s) +
// 85 (1500-3500ms) windows accommodate this floor.
let _WAIT_INT32 = null;
let _SAB_FALLBACK_LOGGED = false;
try {
  const _WAIT_SAB = new SharedArrayBuffer(4);
  _WAIT_INT32 = new Int32Array(_WAIT_SAB);
} catch {
  // SAB unavailable; lock-wait will fall back to busy-wait (see below).
  // _WAIT_INT32 stays null as fallback signal.
}

// architect FLAG-3: Atomics.wait return-value observability. For this
// usage (always wait on int32[0]==0 with no Atomics.notify call ever),
// only 'timed-out' or 'not-equal' should occur. 'ok' would mean
// Atomics.notify was called — impossible by design. Log once per process
// if return value anomalous (defense-in-depth per ADR-0001 invariant 3).
let _UNEXPECTED_WAIT_RESULT_LOGGED = false;

function _waitSleep(sleepMs) {
  // code-reviewer HIGH-CR1 LIVE BUG absorption: Atomics.wait(NaN) blocks
  // process indefinitely per ECMA-262 §25.4.5 (ToNumber(NaN)=NaN treated
  // as +Infinity). Current busy-wait silently no-ops on NaN. Bounds-check
  // preserves prior no-hang behavior under exotic input.
  const safeSleepMs = (typeof sleepMs === 'number' && sleepMs > 0 && isFinite(sleepMs))
    ? sleepMs : 50;
  if (_WAIT_INT32) {
    const result = Atomics.wait(_WAIT_INT32, 0, 0, safeSleepMs);
    if (result !== 'timed-out' && result !== 'not-equal' && !_UNEXPECTED_WAIT_RESULT_LOGGED) {
      _UNEXPECTED_WAIT_RESULT_LOGGED = true;
      try {
        // Defensive observability; never throw from logger path.
        process.stderr.write(`[_lib/lock] unexpected Atomics.wait result: ${result}\n`);
      } catch { /* stderr write failed; ignore */ }
    }
  } else {
    // Fallback busy-wait when SAB unavailable (architect FLAG-1 absorption).
    // Log once per process for ops visibility.
    if (!_SAB_FALLBACK_LOGGED) {
      _SAB_FALLBACK_LOGGED = true;
      try {
        process.stderr.write('[_lib/lock] SharedArrayBuffer unavailable; falling back to busy-wait\n');
      } catch { /* stderr write failed; ignore */ }
    }
    const end = Date.now() + safeSleepMs;
    while (Date.now() < end) { /* spin */ }
  }
}

function acquireLock(lockPath, opts) {
  // ... existing setup (mkdir, maxWaitMs, sleepMs, start) ...
  while (Date.now() - start < maxWaitMs) {
    try {
      fs.writeFileSync(lockPath, String(process.pid), { flag: 'wx' });
      return true;
    } catch {
      // ... existing stale-lock-recovery (preserved) ...

      // H.9.10: replace H.9.7 busy-wait spin loop with Atomics.wait true-sleep
      // primitive (closes drift-note candidate referenced at H.9.7 comment).
      // _waitSleep encapsulates: NaN-guard + Atomics.wait happy path + busy-wait
      // fallback (if SAB unavailable) + observability for unexpected wait results.
      _waitSleep(sleepMs);
    }
  }
  return false;
}
```
