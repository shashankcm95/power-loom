# H.9.9 — error-critic.js fail-soft contract upgrade

## Context

`hooks/scripts/error-critic.js` is a PostToolUse hook (256 LoC) that escalates repeated same-command failures via a forcing instruction. At H.7.10 (CS-2 mira C-2 race fix), the read-modify-write (RMW) at L209-239 (count file + log file updates) was wrapped in `withLock(LOCK_PATH, …)` from `scripts/agent-team/_lib/lock.js` to close a concurrent-PostToolUse-fires race.

**The fail-soft contract gap**: `withLock` from `_lib/lock.js` at L86-92 calls `process.exit(2)` on lock-acquisition timeout (3000ms default). error-critic.js is a hook → ADR-0001 invariant 2 binds: `Hooks that fail return decision: approve (PreToolUse) or exit cleanly (other lifecycles); never block on hook errors`. Lock-acquisition timeout IS a hook error. The current code violates ADR-0001.

**The outer try-catch at L192-256 does NOT catch `process.exit(2)`** (Node's process.exit bypasses try/catch). The fail-open envelope at L251-255 (`logger('error', { error: err.message })`) only catches JS exceptions, not exit calls. The 3-tier `withLock` loading fallback at L57-72 only catches MISSING primitive, not timeout-of-acquired-primitive.

**HT.2.3 Part B precedent**: same-class fix landed for `session-end-nudge.js` at HT.2.3 (commit pattern at `0e50a46`-cohort). Migrated from inline lock primitive → `_lib/lock.js` `acquireLock` + `releaseLock` primitives (which DO return false-on-timeout). Hook fail-soft contract preserved via explicit if/return at call site. HT.2.3 architect MEDIUM-A4 absorption explicitly identified error-critic.js as the deferred sibling: `error-critic.js is already on the _lib/lock.js consumer surface; the deferred work is a fail-soft contract upgrade (migrate from withLock to acquireLock+releaseLock primitives so it doesn't exit-2 on timeout)`.

**Why MANDATORY per-phase pre-approval gate** (per ADR-0002 + ADR-0001):
- Touches load-bearing ADR-0001 invariant 2 (hook fail-soft contract)
- Substrate-fundament-touching (hook subsystem; user-session impact if buggy)
- Option-axis design surface (LOCK_TIMEOUT_MS value; behavior on missing-primitive + timeout-of-acquired-primitive)
- Test surface expansion (new lock-contention test required to validate fail-soft path)
- Per-phase pre-approval gates have caught HIGH-class bugs in 6 of last 8 substrate `_lib/*` migrations (HT.1.8, HT.1.13, HT.2.3, H.9.7, H.9.8)

### `_lib/lock.js` API surface

- `acquireLock(lockPath, opts)` L34-77: returns `true` on success, `false` on timeout; opts.maxWaitMs default 3000ms; opts.sleepMs default 50ms; auto-creates parent dir (HT.2.3 Part A); PID-based stale recovery

- `withLock(lockPath, fn, opts)` L86-92: calls `acquireLock` + on failure `console.error + process.exit(2)`; on success `try fn() finally releaseLock`

### ADR-0001 invariants (substrate hook fail-soft contract)

1. Every hook has a top-level try/catch that prevents uncaught exceptions
2. Hooks that fail return decision: approve (PreToolUse) or exit cleanly (other lifecycles); never block on hook errors
3. Every fail-open path goes through `logger('error', ...)` so the failure is observable in `~/.claude/logs/<hook>.log`
4. Hard-block decisions (PreToolUse decision: block) are reserved for genuine policy violations, not hook errors

Invariant 2 is the load-bearing one for H.9.9: error-critic.js currently fails invariant 2 on lock-acquisition timeout.

### HT.2.3 Part B migration pattern (reference shape)

## Option-axis design surface

### Option A — `acquireLock` + `releaseLock` primitives drop-in (HT.2.3 Part B precedent; APPROVED at gate)

**Implementation** (post-architect HIGH-1 + code-reviewer HIGH-CR2 convergent absorption):
- Replace 3-tier `withLock` loading with 3-tier `{ acquireLock, releaseLock }` destructure loading
- **Fallback semantic INVERTED per architect HIGH-1**: on missing primitive, `acquireLock = () => false` (fail-soft skip RMW) NOT `() => true` (proceed unlocked — would silently regress C-2 race protection from H.7.10)
- **typeof-guard added per code-reviewer HIGH-CR2**: explicit `typeof acquireLock !== 'function'` check post-destructure to force fallback catch if `_lib/lock.js` is structurally broken (e.g., exports `{ withLock }` only without `acquireLock`/`releaseLock`)
- Replace `withLock(LOCK_PATH, fn)` call site with explicit `acquireLock + try/finally releaseLock` pattern

```javascript
let acquireLock, releaseLock;
try {
  ({ acquireLock, releaseLock } = require('../../scripts/agent-team/_lib/lock'));
  // HIGH-CR2 typeof-guard: catch structurally-broken _lib/lock.js (silent
  // undefined-vars from incomplete destructure) before downstream TypeError.
  if (typeof acquireLock !== 'function' || typeof releaseLock !== 'function') {
    throw new Error('_lib/lock.js API shape mismatch — missing acquireLock or releaseLock export');
  }
} catch {
  try {
    ({ acquireLock, releaseLock } = require(path.join(os.homedir(), '.claude', 'scripts', 'agent-team', '_lib', 'lock')));
    if (typeof acquireLock !== 'function' || typeof releaseLock !== 'function') {
      throw new Error('_lib/lock.js API shape mismatch');
    }
  } catch {
    // HIGH-1 fail-soft-skip semantic (NOT proceed-unlocked):
    // acquireLock = () => false skips RMW entirely on missing primitive;
    // strictly better than unlocked-RMW which would silently regress the
    // H.7.10 mira C-2 race fix. Loses escalation for this firing; preserves
    // observability via lock_primitive_missing log.
    acquireLock = () => false;
    releaseLock = () => {};
    logger('lock_primitive_missing', {
      tried: ['../../scripts/agent-team/_lib/lock', '~/.claude/scripts/agent-team/_lib/lock'],
      fallback: 'fail-soft-skip',
    });
  }
}

// ... at the RMW site (replaces lines 209-239 outer withLock):
const haveLock = acquireLock(LOCK_PATH, { maxWaitMs: LOCK_TIMEOUT_MS });
if (!haveLock) {
  // ADR-0001 invariant 2: hook fail-soft on lock contention. Silent-skip vs
  // stale-count emission decided silent-skip per architect MED-3 absorption
  // — emitting forcing instruction with stale count would either undercount
  // (skip increment) or risk corruption (write without lock). The
  // lock_timeout log event IS the observability path per ADR-0001
  // invariant 3 (`Every fail-open path goes through logger so failure is
  // observable in ~/.claude/logs/<hook>.log`).
  logger('lock_timeout', { timeout_ms: LOCK_TIMEOUT_MS });
  return;
}
try {
  // ... RMW (count file read+update; log file append+trim)
} finally {
  releaseLock(LOCK_PATH);
}
```

**Pros**:
- Mirrors HT.2.3 Part B precedent exactly (Option B2 selected over Option B1 `onLockTimeout` callback per YAGNI; 2-consumer-deep is still YAGNI territory)
- Hook fail-soft contract becomes explicit at call site (visible in error-critic.js itself per ADR-0001 institutional concern)
- PID-based stale recovery preserved (strictly better than inline alternatives)
- Drop-in scope: existing `_lib/lock.js` primitives sufficient; no new API surface
- Symmetric fallback shape: no-op for missing primitive mirrors withLock's no-op fallback (same lock-not-available degradation profile)

**Cons**:
- Call site grows from ~30 LoC (single withLock block) to ~40 LoC (if/return + try/finally)
- 3-tier fallback duplicates session-end-nudge.js shape (DRY candidate but bounded to 2 consumers; defer per YAGNI)
- LOCK_TIMEOUT_MS introduces tunable; need explicit value choice

### Option B — Extend `withLock` with `onLockTimeout` callback (2nd-consumer reconsidered)

**Implementation**: extend `withLock` API to accept optional `onLockTimeout` callback; falls back to current exit-2 behavior if callback absent; error-critic.js passes a callback that logs `lock_timeout` + returns silently.

```javascript
// _lib/lock.js
function withLock(lockPath, fn, opts) {
  if (!acquireLock(lockPath, opts)) {
    const onTimeout = opts && opts.onLockTimeout;
    if (typeof onTimeout === 'function') {
      return onTimeout({ lockPath, maxWaitMs: (opts && opts.maxWaitMs) || 3000 });
    }
    console.error(`Could not acquire lock at ${lockPath} within ${(opts && opts.maxWaitMs) || 3000}ms. Aborting.`);
    process.exit(2);
  }
  try { return fn(); } finally { releaseLock(lockPath); }
}

// error-critic.js
withLock(LOCK_PATH, () => { /* RMW */ }, {
  maxWaitMs: LOCK_TIMEOUT_MS,
  onLockTimeout: () => {
    logger('lock_timeout', { timeout_ms: LOCK_TIMEOUT_MS });
  },
});
```

**Pros**:
- 2 hook consumers (session-end-nudge.js + error-critic.js) now both want fail-soft → reaches YAGNI threshold (`>=2 consumers want it` per HT.1.x heuristic)
- ADR-0001 invariant 2 + ADR-0003 fail-soft contract becomes parameterizable in `_lib/*` API
- Hook code becomes denser (no explicit if/return; callback-driven)
- Future hooks can adopt `withLock + onLockTimeout` directly

**Cons**:
- API surface change to `_lib/lock.js` (new opt); 11 substrate `withLock` consumers must be reviewed for cascade impact (verified none require this option; all 9 non-hook consumers use substrate-script semantics where exit-2 is correct)
- Hook contract becomes implicit-via-callback (less visible per ADR-0001 institutional concern about explicit-at-call-site fail-soft)
- session-end-nudge.js migrated to Option B2 at HT.2.3 — if H.9.9 adopts B1, session-end-nudge.js becomes inconsistent unless retrofitted (out of H.9.9 scope but cohort-aesthetic concern)
- Introduces asymmetry: timeout handler runs OUTSIDE the try/finally → no releaseLock attempted (correct since never acquired; but conceptually muddles "withLock" semantics)

### Recommendation: Option A

Rationale:
1. **Consistency with HT.2.3 Part B precedent**: same hook fail-soft contract upgrade pattern; substrate-internal convention alignment
2. **Explicit-at-call-site fail-soft per ADR-0001**: institutional concern is fail-soft logic must be visible in hook itself, not abstracted behind API toggle
3. **YAGNI bound holds for now**: 2 hook consumers, but session-end-nudge.js already adopted Option B2; promoting to Option B1 would require retrofitting session-end-nudge.js (cohort scope creep)
4. **No `_lib/lock.js` API surface change**: lower-blast-radius migration (only error-critic.js changes); preserves audit trail for 11 existing consumers
5. **Future-revisit trigger**: if a 3rd hook adopts the same pattern, revisit `onLockTimeout` API extension (~2026-Q3 candidate per HT.2.3 architect MEDIUM-A1 deferred-CPU-cost pattern)

## LOCK_TIMEOUT_MS selection

PostToolUse hooks fire after every Bash tool execution. Contention rare in single-agent flow; possible in multi-agent HETS flows where parallel subagent Bash calls bunch up.

- **Too short** (e.g., 500ms): false-positive timeouts under normal contention → fail-soft skips → escalation undercount → loses C-2 race fix value
- **Too long** (e.g., 5000ms): perceivable hook delay → user notices unresponsive sessions
- **HT.2.3 Part B precedent for session-end-nudge.js**: 2000ms (Stop hook; lower-contention than PostToolUse)
- **Recommendation**: 2000ms — matches HT.2.3 Part B; conservative; acceptable hook delay; allows for substrate-script-typical contention bursts

Gate may surface alternative value with rationale.

### Tests added (post-architect HIGH-2 + code-reviewer HIGH-CR3 + MED-CR4 + MED-CR6 convergent absorption)

**`tests/smoke-h7.sh`** — append Test 14 (cohort with Tests 11-13). Test shape revised per gate FLAGs:
- Stale-PID determinism via parent shell's $$ PID (guaranteed-alive parent process; deterministic across macOS/Linux; mirrors HT.2.3 Test 79 approach)
- `trap EXIT` for reliable cleanup if test errors out mid-run (per code-reviewer MED-CR6)
- Explicit `grep -q 'lock_timeout'` assertion on `error-critic.log` per ADR-0001 invariant 3 observability requirement (code-reviewer HIGH-CR3)
- Tighter elapsed-time window via Node millisecond timing (per code-reviewer MED-CR4); accept 1500-3500ms envelope (~LOCK_TIMEOUT_MS=2000ms ± 750ms tolerance)

```bash
# Test 14 (H.9.9): error-critic.js lock-contention fail-soft contract.
# Validates ADR-0001 invariant 2 (hooks never block on hook errors) + invariant 3
# (every fail-open path goes through logger so failure is observable).
# Approach: write parent shell's $$ PID (guaranteed-alive) into LOCK_PATH;
# acquireLock probes via process.kill(pid, 0) — pid is alive so probe succeeds
# → wait loop continues → 2000ms LOCK_TIMEOUT_MS expires → acquireLock returns
# false → error-critic.js logs lock_timeout + returns silently (fail-soft).
h99_session="h99-test-$$"
h99_failure_dir="$TMPDIR/.claude-toolkit-failures/$h99_session"
h99_log="$HOME/.claude/logs/error-critic.log"
mkdir -p "$h99_failure_dir"
# Use parent shell PID — guaranteed alive for the test's duration (macOS + Linux portable)
echo "$$" > "$h99_failure_dir/.lock"
# trap cleanup per code-reviewer MED-CR6: ensures lock file removed even if test errors mid-run
trap 'rm -f "$h99_failure_dir/.lock" 2>/dev/null' EXIT
# Capture log size pre-test for delta probe (avoids false positive from prior lock_timeout events)
h99_log_size_before=$(stat -f %z "$h99_log" 2>/dev/null || stat -c %s "$h99_log" 2>/dev/null || echo 0)
# Run with millisecond-resolution timing via Node Date.now()
h99_start_ms=$(node -e "console.log(Date.now())")
H99_EXIT=0
h99_result=$(echo '{"tool_name":"Bash","tool_input":{"command":"npm test"},"tool_response":{"stderr":"Error: failed","is_error":true}}' | CLAUDE_SESSION_ID="$h99_session" node "$CLAUDE_DIR/hooks/scripts/error-critic.js" 2>/dev/null) || H99_EXIT=$?
h99_end_ms=$(node -e "console.log(Date.now())")
h99_elapsed_ms=$((h99_end_ms - h99_start_ms))
rm -f "$h99_failure_dir/.lock"
trap - EXIT
# Probe (a): exit code 0 (fail-soft; NOT 2 from old withLock path)
# Probe (b): stdout empty (no forcing instruction emitted; RMW skipped)
# Probe (c): elapsed time in 1500-3500ms envelope (LOCK_TIMEOUT_MS=2000ms ± 750ms)
# Probe (d): lock_timeout log event appended to error-critic.log per ADR-0001 invariant 3
h99_log_size_after=$(stat -f %z "$h99_log" 2>/dev/null || stat -c %s "$h99_log" 2>/dev/null || echo 0)
h99_log_grew=$((h99_log_size_after > h99_log_size_before ? 1 : 0))
h99_log_has_timeout=$(tail -c $((h99_log_size_after - h99_log_size_before + 100)) "$h99_log" 2>/dev/null | grep -c 'lock_timeout' || echo 0)
if [ $H99_EXIT -eq 0 ] && [ -z "$h99_result" ] && [ $h99_elapsed_ms -ge 1500 ] && [ $h99_elapsed_ms -le 3500 ] && [ $h99_log_grew -eq 1 ] && [ $h99_log_has_timeout -ge 1 ]; then
  echo "  ✓ error-critic: H.9.9 lock-contention fail-soft contract (exit=0, no forcing instruction, elapsed=${h99_elapsed_ms}ms, lock_timeout logged)"
  passed=$((passed + 1))
else
  echo "  ✗ error-critic: H.9.9 lock-contention fail-soft — exit=$H99_EXIT result='${h99_result:0:60}' elapsed=${h99_elapsed_ms}ms log_grew=$h99_log_grew log_has_timeout=$h99_log_has_timeout"
  failed=$((failed + 1))
fi
```
