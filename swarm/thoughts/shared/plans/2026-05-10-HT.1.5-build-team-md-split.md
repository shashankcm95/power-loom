# Phase HT.1.5 — `commands/build-team.md` 322 LoC + 6-responsibility split (sibling decision under ADR-0002)

## Context

`commands/build-team.md` is the chat-agent-followed slash command that orchestrates HETS team spawn. HT.0.3 audit ranked it B.1 + C.1 (322 LoC; ~45% embedded bash claimed) on the multi-responsibility-at-lifecycle-entrypoint pattern. HT.1 backlog entry HT.1.5 (lines 139-149) prescribes "extract bash; commands/build-team.md becomes orchestrator narrative with helper-script invocations."

**Sibling decision under ADR-0002**: third application of the bridge-script entrypoint criterion (HT.1.3 was first; HT.1.4 was second). Per ADR-0002 cross-language application, markdown post-split shape is "orchestrator narrative + helper-script invocations" — distinct from HT.1.3's Node.js dispatcher pattern and HT.1.4's bash sourced-file decomposition. Same criterion (lifecycle coherence + ≤800 LoC AND ≤5 responsibilities); third post-split shape.

**Why ADR-0002 fires for HT.1.5**: 322 LoC is well under the 800 LoC bound BUT 6 responsibilities > 5 responsibilities bound. Responsibility-count is the primary trigger (per ADR-0002 Decision: "EITHER size bound is breached"). Lifecycle coherence holds (single `/build-team <task>` invocation; chat-agent-followed orchestration in sequence).

**Dispatch idiom** (HT.1.5-verify code-reviewer Q3 MEDIUM specification): bash `case "$1" in` positional dispatch with `--help` handled BEFORE the case block. Unknown subcommand exits 1 with usage on stderr. Pattern:

```bash
case "$1" in
  --help|-h)
    usage
    exit 0
    ;;
  route-decide-gate)
    shift
    cmd_route_decide_gate "$@"
    ;;
  build-spawn-context)
    shift
    cmd_build_spawn_context "$@"
    ;;
  verify-with-contract-selection)
    shift
    cmd_verify_with_contract_selection "$@"
    ;;
  assign-challenger-pair)
    shift
    cmd_assign_challenger_pair "$@"
    ;;
  record-verdict)
    shift
    cmd_record_verdict "$@"
    ;;
  *)
    echo "Unknown subcommand: $1" >&2
    usage >&2
    exit 1
    ;;
esac
```

**Path resolution** (HT.1.5-verify code-reviewer Q4 HIGH + architect Q6 LOW; converged): all helper invocations use `$HOME/Documents/claude-toolkit/...` matching the existing `ROUTE_DECIDE_SCRIPT="$HOME/Documents/claude-toolkit/scripts/agent-team/route-decide.js"` pattern at build-team.md line 32. **Original draft used `$SCRIPT_DIR` which is NOT set in slash-command context** — confirmed by exhaustive grep across all 14 commands; no command sets or references `$SCRIPT_DIR`. Using $SCRIPT_DIR would have caused runtime failure (helper script not found). HOME-relative path matches the established pattern across every slash command. **HIGH severity correction absorbed.**

## CLI surface preservation strategy

**External consumers**:
- `/build-team <task>` slash command invocation (chat-agent-followed) — preserved (`commands/build-team.md` still the entry point; chat agent reads narrative + invokes helper)
- Helper script invocation pattern matches existing `_lib/*` precedent for substrate-internal utilities (no breaking change visible to slash-command users)

**Internal flow**:
- Chat agent reads commands/build-team.md narrative → understands semantics
- Chat agent invokes `bash $SCRIPT_DIR/scripts/agent-team/build-team-helpers.sh <subcommand> <args>` per Step
- Helper script returns JSON (or text) to stdout; chat agent parses and continues per narrative description
- spawn_implementer / spawn_challenger placeholders remain inline — these are chat-agent-actions (Agent tool invocations), NOT bash. Helper script does NOT replace these.

**Important non-extraction**: `spawn_implementer` and `spawn_challenger` (per kb:hets/spawn-conventions and kb:hets/challenger-conventions) are **chat-agent action placeholders** — they tell Claude to use the Agent tool with specific persona/contract/skills configuration. These are NOT bash; they remain in the markdown narrative. The helper script wraps SUBSTRATE PRIMITIVE calls (route-decide.js, contract-verifier.js, agent-identity.js, pattern-recorder.js); it does NOT wrap chat-agent actions.
