# HT.3.1 — ADR tier taxonomy codification

## Context

Post-HT audit-followup (5-agent chaos test + HETS code review of HT.x cumulative state) surfaced a schema-vs-prose mismatch: ADR-0005 lines 60-66 explicitly declare a three-tier ADR taxonomy (technical / governance / editorial) and map each existing ADR to a tier, but the ADR frontmatter schema has no `tier` field. The taxonomy is operational in prose but not machine-readable.

Per ADR-0005 lines 60-66 (verbatim quote):

> **ADR taxonomy positioning** (post-HT.1.7 + HT.1.13 architect MEDIUM-2 absorption): the substrate now operates a three-tier ADR taxonomy:
>
> - **Technical-tier** ADRs (e.g., ADR-0001 — fail-open hook discipline) — codify MECHANICAL invariants verifiable by grep/lint/test (try/catch + logger + decision: approve + hard-block scope). Drift detection is mechanical.
> - **Governance-tier** ADRs (e.g., ADR-0003 — institutional commitment to enforce ADR-0001) — codify INSTITUTIONAL commitments + load-bearing code-review gates. Drift detection is human review of process discipline.
> - **Editorial-tier** ADRs (this one — slopfiles authoring discipline) — codify AUTHORING discipline that depends on best-effort instruction-following (LLM-side) + author-side curation. Drift detection is partition-decision review at PR time.

The audit's reframing: codify the taxonomy at the schema layer (frontmatter `tier` field) so the existing prose mapping becomes machine-readable. This is the smallest schema change that closes the schema-vs-prose gap.

**Why now (HT.3.1 timing)**: Tier 2 institutional reframing is the natural follow-up to Tier 1 mechanical fixes (shipped at `a00aeaa` on 2026-05-11). Codifying the taxonomy first (HT.3.1) makes ADR-0002 status flip (HT.3.3) cleaner — it inherits a `tier: technical` annotation simultaneously with the status transition.

**Why sub-plan-only methodology** (per HT.1.6 decision-rationale-matrix convention):

| Trigger | Status | Reasoning |
|---------|--------|-----------|
| Fresh design surface | NO | Tier taxonomy + classifications already declared in ADR-0005 prose at HT.1.13 |
| Schema change | YES (minor) | Additive frontmatter field; no existing consumer reads frontmatter beyond `adr.js` list/read which is forward-compatible |
| Option-axis decision | NO | Each existing ADR's tier classification is unambiguous from its content + ADR-0005 prose mapping |
| Institutional discipline encoding | YES (codification) | But codification of EXISTING prose discipline, not NEW commitment — ADR-0005 already operates the discipline |
| HIGH-class bug catchable at design | NO | Mechanical Edit operations on frontmatter; no runtime behavior change; existing tests are agnostic to the field |

3 of 5 triggers absent; 2 present but in codification-only form (existing discipline). Net: sub-plan-only with explicit decision-rationale matrix, matching HT.2.4 (mechanical doc-lag cleanup) + HT.1.7 schema-extension precedents.

Per dominant-invariant disambiguation rule, governance wins.

- **Closes audit finding cleanly**: 5-agent chaos test + HETS code review surfaced "schema-vs-prose gap" — ADR-0005 declares taxonomy in prose; HT.3.1 codifies in frontmatter

- Cross-substrate adoption: this taxonomy applies to substrate ADRs only; downstream consumers can adopt or not at their discretion
