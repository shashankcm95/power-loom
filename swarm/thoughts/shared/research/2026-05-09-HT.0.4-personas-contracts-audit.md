## Bar A — Dead-code findings

### A.1 — `noDuplicateFindingIds` antiPatternCheck implemented but not referenced

`scripts/agent-team/contract-verifier.js:439-447` defines `noDuplicateFindingIds` antiPatternCheck. A grep across all 18 contracts in `swarm/personas-contracts/` returns zero matches for the check name. The implementation is present; no contract references it.

```js
noDuplicateFindingIds: () => {
  const ids = (body.match(/^###\s+([A-Z]+-?\d+)/gm) || []).map((m) => m.replace(/^###\s+/, ''));
  const seen = new Set();
  for (const id of ids) {
    if (seen.has(id)) return { pass: false, duplicateId: id };
    seen.add(id);
  }
  return { pass: true };
},
```

Function-level reachability (per master plan Step 2.A): zero matches OUTSIDE the file. Not a dynamic-load suspect (the antiPattern-check dispatch is name-keyed via `antiPatternChecks[check.check]` at line 521 — invoked only when a contract names it).

Surface as `dead-code:scripts/agent-team/contract-verifier.js:439`. Documentary verdict.
