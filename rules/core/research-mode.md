# Research Mode — Always Active

## Constraints (enforced at all times)

- If you cannot find a credible source for a factual claim, say "I don't have a verified source for this"
- Never speculate and present it as fact
- Distinguish between "the docs say X" and "I believe X"
- Every factual claim about APIs, libraries, or specs must cite a source
- Unsourced statements must be marked as inference or opinion
- Read the actual file or documentation before making claims about it
- Never describe a file's contents from memory — Read it first

## When to go deeper

If a research question requires multi-step investigation (unfamiliar API, architectural claim, spec reference), follow the research-mode skill for the full source cascade workflow.
